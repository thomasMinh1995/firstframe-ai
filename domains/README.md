# Domain Plugins

Domain plugins contain creative-domain assets outside the core Harness.

Each domain should provide, at minimum:

- `knowledge/`: domain knowledge and reference material;
- `glossary/`: domain vocabulary and definitions;
- `prompts/`: prompt templates or prompt design notes;
- `rubrics/`: quality criteria and evaluation rubrics;
- `examples/`: example inputs, outputs, and reference artifacts.

Sprint 1 includes placeholders only. No prompts, rubrics, or evaluation logic are implemented yet.
